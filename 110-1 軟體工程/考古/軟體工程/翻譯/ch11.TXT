User Interface Design

Interface Design
典型的設計錯誤
1.缺乏一致性
2.太多記憶
3.沒有指導/幫助
4.沒有內容敏感性
5.反應差
6.不友好

Golden Rules
1.用戶置於控制之下
2.減少用戶的記憶體負荷
3.使介面一致

Place the User in Control
1.以不強制使用者進行不需要的動作的方式定義互動模式
2.彈性的互動
3.互動可以中斷或未完成
4.減化互動&允許自定義
5.隱藏臨時用戶的技術內部
6.設計指展現在螢幕的直接互動

Reduce the User’s Memory Load
1.減少短期記憶的需求
2.有意義的默認值
3.定義直觀的快捷
4.視覺布局基於真實世界的比喻
5.漸進方式透露訊息

Make the Interface Consistent
1.允許用戶將目前task放進有意義的內文
2.在一系列應用中維持一致性
3.過去符合期望的互動模型不需更改，除非有強制的理由

★User Interface Design Models
1.使用者模型－系統的所有專端用戶的數據
2.設計模型－用戶模型的設計實現
3.心理模型(系統感知)－用戶對介面的內心想像
4.實施模型－介面"外觀和感覺"連接描述語法語義的支持訊息

Interface Analysis
1.透過介面和系統互動的人
2.終端用戶要完成的任務
3.呈現部分介面的內容
4.引導任務的環境

User Analysis

Task Analysis and Modeling
1.問以下問題
	用戶在特定情況下執行什麼工作?
	用戶工作時執行什麼任務和子任務？
	用戶工作時操作什麼特定問題域物件？
	工作任務的順序是什麼？
	任務的層次是什麼？
2.使用者案例定義基本互動
3.任務訂製詳細互動任務
4.物件定義識別互動物件(class)
5.工作流分析定義涉及多人時工作流程如何完成

Analysis of Display Content
1.不同類型的資料分配給螢幕上固定的位置？
2.用戶可自訂螢幕位置的內容？
3.適當螢幕識別分配給所有內容？
4.大型報告，如何劃分以利理解？
5.如何使用顏色加強理解？
6.如何向用戶顯示錯誤訊息和警告？

Interface Design Steps
1.定義介面物件&操作
2.定義導致介面狀態改變的事件(用戶動作)
3.描述每個介面狀態
4.標明用戶如何過介面提供的訊息來解釋系統狀態

★Design Issues
1.反應時間
2.幫助設施
3.錯誤處理
4.點單和命令標籤
5.應用程序可理解性
6.國際化

WebApp Interface Design
1.我在哪？介面應提供經常訪問的WebApp指示
2.我可以做什麼？介面幫助用戶了解當前選項(可用功能、link)
3.我在哪，要去哪？介面要方便導航(地圖、在WebApp移動的路徑)

Effective WebApp Interfaces
1.有效的介面視覺上是明顯和寬容的
2.有效的介面不涉及用戶系統的內部工作
3.有效的應用和服務執行最大工作量

Interface Design Principles
1.預期－設計為預期使用者的下一步
2.溝通－應該傳達任何用戶發起的活動狀態
3.一致性－使用導航控制、選單、icon、美觀
4.自主性控制－以利用戶在WebApp中操作，
5.效率－設計應優化用戶的共作效率
6.焦點－應專注於用戶手上的任務
7.Fitt定律－"獲取目標的時間是距離、大小的function"
8.Human介面object
9.延遲縮減－多任務處理的方式
10.可用性－設計成最小量學習時間
11.維持產品完整性－自動保存產品，避免丟失
12.可讀性－所以訊息都可以給任何人讀
13.追蹤狀態－追蹤和儲存用戶互動狀態
14.可見導航－

Interface Design Workflow
1.分析模型中的訊息，根據需要細化
2.介面布局的粗略草圖
3.目標映射到特定介面操作
4.定義每個操作相關的用戶任務
5.美學設計優化介面布局
6.辨識實現介面所需的用戶介面物件
7.開發用戶和介面的互動過程
8.開發介面的行為表示
9.描述介面佈局的美的狀態
10.細化&檢查介面式設計模型

Aesthetic Design
1.不要害怕空白
2.強調內容
3.從右上到左下角整理佈局元素
4.不要使用滾動擴展
5.考慮解析度、瀏覽器大小
